PreferredProgrammePlan(Int32, Int32, UvAInfo)

In case a student has multiple main programmes it is sometimes necessary to force a decision which is the "preferred" (most important) one. This cannot be more than a gamble. Here we take the programme that has the longest uninterrupted duration, as long as we limit the coice between programmes in the same career.

Implementation note: this can only be asked for 1 student at a time otherwise the query performance becomes really bad. This seems to be caused by stacking two "GroupBy" clauses:

  • the 1 in "ActiveStudentPrograms"
  • the 1 GroupBy(sp => sp.StudentID) we would need here if we did not select a single student

PreferredProgrammePlan(Int32, UvAInfo)

In case a student has multiple main programmes it is sometimes necessary to force a decision which is the "preferred" (most important) one. This cannot be more than a gamble. Here we first take the programme that has the highest career, and in case of equal careers we take the one with the longest uninterrupted duration.
