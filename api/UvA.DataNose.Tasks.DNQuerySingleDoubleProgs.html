Class DNQuerySingleDoubleProgs

This class knows about single and double programmes, for purposes of knowing the starters of a given programme and year.

Two programmes together may cooperate to offer a double programme. When students are enrolled in both programmes they can get permission to follow the double programme. From then on, the double programme acts much like a separate programme: it has its own list of starting students.

This class makes it possible to treat single and double programmes much alike:

  • You can ask for the starters of an arbitrary programme (single or double)
  • In case of double: only starters are returned that have permission for the double programme AND are still enrolled in both parent programmes.
  • In case of single: only starters are shown that attend this programme as a single programme; i.e. students starting in one of the double programmes that this programme is parent of, are omitted. (I now made this optional)

Use it e.g. to send a mailing where admitted starters for a double programme get a different letter from those that will start in one (or both) of the two parent programmes, without being admitted to the double prog. (The same setup could be used to get lists of participating, not just starting students, but...)

Other, derived questions that this class answers:

  • Given a student enrolment for a parent program: give an array of double programs (children of this parent program) that the student attends.
  • Given a student ID and a (parent) programme name: same as previous
  • Same as previous, but return only the first double programme name (or empty string if there are none)
